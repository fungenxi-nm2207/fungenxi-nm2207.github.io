Problems faced: To add interactivity to the game, I have decided to add a user prompt using Javascript, but that function was not showing up correctly. I thought that it was due to the 'prompt' function directly clashing with the 'addEventListener' function that I added to listen for players' clicks to show the hidden page. After talking to Wien about my problem during class, the solution was to add a 'id= introduction' next to my 'div class= introduction'. After reflecting, I realised the problem.

In the JS prompt code, I indicated 'introduction.innerHTML= "hey"... ' The system assumed 'introduction' to be a reference to an HTML element with an ID of 'introduction', in which it will then set the content of that element to the string. By putting only 'div class= introduction' it was not able to 'getElementById' nor use the 'innerHTML' property.

Q1: What does a game do?
In my game, players will realise that they are trapped in "the box", where the only way to get out is to complete challenges and solve mysteries along the way. Challenges would include dodging obstacles, clicking the right boxes at the right time, and maybe some puzzles, that require users to think out of the box!

Q2: What are the rules it follows?
#Challenge 1: The Box Dodger
The player must control a character using the arrow keys (left and right) to avoid the falling boxes. These boxes might fall at different timings, so it is up to the players to react in time. If the character did not manage to dodge the box in time, the challenge is failed and they must restart again. The player must complete the challenge within a certain time limit to progress to the next challenge.

#Challenge 2: Seize the opportunity
Player is presented with an empty screen, and have to wait for the boxes to appear and clicking on it before it disappears, within a set target. The players will be presented with other shapes as well, to increase the difficulty level. If the player clicks on another shape (e.g triangle), the player must restart the challenge.

Q3: What can the user do?
They are forced to restart each challenge until they succeed. They can control their arrows and cursors when playing the game.

Q4: What does the program do for each user action?
Begin game button click: When user clicks the button, the 'startbutton' element is selected when using JS's 'getElementById()' method. An event listener is added to the 'startbutton' element using the 'addEventListener()', which listens for a "click" event. When the user clicks the button, the event listener triggers a function that adds the 'hidden' CSS class to the 'startPage' element and removes the 'hidden' class from the 'gamePage1' element using 'classList.add' and 'classList.remove'. The same mechanic is applied for all of the buttons, when navigating to the next few game pages. However, for the actual game pages, I would put it such that the buttons are only visible after the players complete the challenges.

Box dodger gameplay: Concepts like arrays and conditional statements ('if') are used. Power-ups such as invisibility and additional points are also implemented using JS. The users interact with it by controlling the character with left and right arrows.

Q5: When does the game end?
It ends when the player successfully completes all the challenges and reaches the end of the game. At the end of the game, there will be a mini interactive statement between me and the user; something similar to "why are you still here?". They are then able to decide whether to restart the game, or exit.
